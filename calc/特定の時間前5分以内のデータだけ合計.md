`now` ではなく、基準にしたい **特定の日時** を用意して、その日時の **5分前〜その日時まで** に入るデータだけ合計します。

```java
LocalDateTime targetTime = LocalDateTime.of(2026, 6, 28, 10, 30);
LocalDateTime fromTime = targetTime.minusMinutes(5);

int total = items.stream()
        .filter(item -> item.getId().equals(targetId))
        .filter(item -> !item.getDateTime().isBefore(fromTime))
        .filter(item -> !item.getDateTime().isAfter(targetTime))
        .mapToInt(Item::getCount)
        .sum();
```

これは次の範囲を対象にします。

```java
targetTime.minusMinutes(5) <= item.getDateTime() <= targetTime
```

つまり、例として `targetTime` が `10:30` なら、

```java
10:25:00 ～ 10:30:00
```

のデータだけ合計します。

---

元のコードのこの部分は、

```java
item.getDateTime().isAfter(now.minusMinutes(5))
```

「現在時刻の5分前より後」という意味でした。

特定の時刻を基準にするなら、こう変えます。

```java
LocalDateTime targetTime = ...;
LocalDateTime fromTime = targetTime.minusMinutes(5);
```

そして範囲判定はこうです。

```java
!item.getDateTime().isBefore(fromTime)
&& !item.getDateTime().isAfter(targetTime)
```

1つの `filter` にまとめるなら、こう書けます。

```java
LocalDateTime targetTime = LocalDateTime.of(2026, 6, 28, 10, 30);
LocalDateTime fromTime = targetTime.minusMinutes(5);

int total = items.stream()
        .filter(item -> item.getId().equals(targetId))
        .filter(item -> {
            LocalDateTime dateTime = item.getDateTime();
            return !dateTime.isBefore(fromTime)
                    && !dateTime.isAfter(targetTime);
        })
        .mapToInt(Item::getCount)
        .sum();
```

実務では、境界を含めたいならこの書き方が安全です。
